# 教育智能助手 - 基于RAG的智能问答系统
该项目是一个基于检索增强生成（RAG）技术构建的教育智能助手，通过向量数据库存储教育类文本数据（以操作系统教材为例构建向量库），结合大语言模型实现智能问答，并提供可视化的Web界面支持知识库上传更新和智能问答交互。

## 项目架构
### 核心功能模块
1. **知识库管理模块**：支持TXT文件上传、文本分割、MD5去重、向量入库，基于Chroma向量数据库存储文本向量。
2. **向量检索模块**：基于嵌入模型生成文本向量，实现相似性检索，返回匹配的教育文档片段。
3. **智能问答模块**：结合对话历史和检索到的知识库内容，通过大语言模型生成精准回答。
4. **Web交互模块**：基于Streamlit实现可视化界面，包含知识库上传页面和智能问答页面。

### 技术栈
- **核心框架**：LangChain（RAG流程编排）、Streamlit（Web界面）
- **向量数据库**：Chroma（本地轻量化向量存储）
- **嵌入模型**：通义千问 text-embedding-v4
- **大语言模型**：通义千问 qwen3-max
- **辅助工具**：hashlib（MD5去重）、datetime（元数据记录）

## 快速开始
### 环境依赖
```bash
# 安装核心依赖（若还有依赖缺失请自行安装）
pip install streamlit langchain langchain-chroma langchain-community langchain-text-splitters
```

### 运行项目
#### 1. 知识库上传服务
```bash
streamlit run app_file_uploader.py
```
- 访问本地地址（默认：http://localhost:8501）
- 上传TXT格式的教育文本（如操作系统教材），系统会自动完成文本分割、向量化并入库（自动去重）。

#### 2. 智能问答服务
```bash
streamlit run app_qa.py
```
- 在聊天界面输入问题，系统会从向量库检索相关教材片段，结合对话历史生成专业回答。

## 核心配置说明
配置文件 `config_data.py` 包含项目关键参数，可根据需求调整：
| 参数名 | 说明 | 默认值 |
|--------|------|--------|
| `collection_name` | Chroma向量库集合名 | rag |
| `persist_directory` | 向量库本地存储路径 | ./chroma_db |
| `chunk_size`/`chunk_overlap` | 文本分割长度/重叠长度 | 800/150 |
| `top_k` | 检索返回匹配文档数 | 3 |
| `embedding_model_name` | 嵌入模型名称 | text-embedding-v4 |
| `chat_model_name` | 大语言模型名称 | qwen3-max |
| `session_id` | 对话会话ID | uuid生成 |

## 核心模块说明
### 1. 知识库处理（knowledge_base.py）
- 文本去重：通过MD5校验避免重复上传相同内容。
- 文本分割：使用`RecursiveCharacterTextSplitter`按自然分隔符分割长文本。
- 向量入库：将分割后的文本片段通过嵌入模型生成向量，存入Chroma数据库，并记录元数据（文件名、创建时间、操作人）。

### 2. 向量检索（vector_stores.py）
- 封装Chroma向量库，提供检索器接口，支持按相似性返回指定数量的文档。

### 3. 对话历史管理（file_history_store.py）
- 基于本地文件存储对话历史，通过`uuid`（后续会优化）区分不同用户会话，支持消息追加、读取和清空。

### 4. RAG流程编排（rag.py）
- 构建RAG链：用户输入 → 向量检索 → 拼接上下文+对话历史 → 大模型生成回答。
- 支持流式输出，适配Web界面的实时响应。

### 5. Web界面（app_file_uploader.py / app_qa.py）
- `app_file_uploader.py`：可视化上传TXT文件，展示文件信息，实时反馈入库状态。
- `app_qa.py`：聊天界面展示对话历史，加载中状态提示，流式输出AI回答。

## 项目特点
1. **轻量化部署**：基于本地Chroma向量库，无需依赖云端数据库，部署简单。
2. **去重机制**：通过MD5校验文本内容，避免重复入库，节省存储和计算资源。
3. **会话记忆**：支持对话历史管理，结合上下文生成更精准的回答。
4. **可视化交互**：Streamlit界面操作简单，无需前端开发经验，快速上手。

## 扩展方向
1. 支持更多文件格式（PDF、Word等）的上传解析。
2. 优化检索策略，增加关键词检索、多维度相似性排序。
3. 扩展多会话管理，支持多用户独立对话。
4. 接入更多大语言模型，支持模型切换和参数调优。
5. 增加知识库内容编辑、删除功能，完善知识库管理能力。

## 注意事项
1. 确保已配置通义千问API密钥（环境变量或配置文件），否则嵌入模型和大语言模型无法调用。
2. 上传的TXT文件建议编码为UTF-8，避免中文乱码。
3. 向量库数据默认存储在`./chroma_db`目录，删除该目录会导致知识库数据丢失。
4. 对话历史默认存储在`./chat_history`目录，按`session_id`分文件存储。
